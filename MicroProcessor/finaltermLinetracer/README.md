## 시행 착오1
- 문제 :
1) 관성으로 인한 이탈
2) 코너 각도를 작게 주어서, 많은 검사 실행
- 해결 방안 :
1) 제동 문제 : 좌우 모터에 backward, forward를 반대로 줘서 코너링 및 접지력 강화
2) 모델 복잡도 : 적당한 코너 각도값을 실험을 통해 파악, 최대한 코드 단순화

## 최종 코드 : 코드 단순화 및 파라미터 변화
- 주안점 : 제동(backward forward 함께 활용), 코드 단순화(코너 각도 넓힘), 속도 증가
- 지속적인 실험을 통한 파라 미터 수정 과정
- 적당한 파라미터 값 파악시 활용한 알고리즘 : 이분 탐색 방법

1) 직진 : 255 (위험) -> 250 안정 -> 252 안정 -> 253(최종)
2) 코너 : 180 으로 양쪽 모두 적용 -> 모터 성능 차이를 고려한 diff 변수 적용(왼쪽 모터) -> 150, 150 수정 -> 180, 150 (최종)
3) diff : 35(이탈 많음) -> 32(조금 적음) -> 30 (최종)
4) 교차로 구간 : 멈추고 직진하도록 초기 구상 -> 그냥 무시하고 직진(최종)
